1. login / signup for Snowflake trial or standard 30 days free trial

2. Open SQL Worksheet and run below command
-- Use the account admin role to set things up
USE ROLE ACCOUNTADMIN;

-- Create a dedicated role for dbt
CREATE ROLE IF NOT EXISTS dbt_role;
GRANT ROLE dbt_role TO USER <student_snowflake_username>;

-- Create a dedicated warehouse
CREATE WAREHOUSE IF NOT EXISTS dbt_wh 
  WITH WAREHOUSE_SIZE = 'XSMALL' 
  AUTO_SUSPEND = 60 
  AUTO_RESUME = TRUE;
GRANT USAGE ON WAREHOUSE dbt_wh TO ROLE dbt_role;

-- Create the database and raw/marts schemas
CREATE DATABASE IF NOT EXISTS dbt_demo_db;
GRANT ALL ON DATABASE dbt_demo_db TO ROLE dbt_role;

USE DATABASE dbt_demo_db;
CREATE SCHEMA IF NOT EXISTS raw;
CREATE SCHEMA IF NOT EXISTS marts;
GRANT ALL ON SCHEMA raw TO ROLE dbt_role;
GRANT ALL ON SCHEMA marts TO ROLE dbt_role;

Click the yellow button to download the latest version for Windows.

Step 2: Run the Installer (The Crucial Step)
Open the downloaded .exe file. Before clicking "Install Now", look at the very bottom of the installer window:

⚠️ CRUCIAL: You must check the box that says "Add python.exe to PATH".

If you do not check this box, Windows will continue to say "python is not available." Once checked, click Install Now.

Run the initialize command:

Bash
dbt init my_snowflake_project

The terminal will ask a series of setup questions. Answer them precisely as follows:

Which database would you like to use? Type 1 or snowflake.

account: Enter the Snowflake Account Identifier. (This is found in the lower-left corner of the Snowflake UI. It looks like xy12345.us-east-1 or abc12345. Do not include snowflakecomputing.com).

user: Enter the Snowflake username.

authentication: Type 1 or password.

password: Enter the Snowflake account password.

role: dbt_role (The role created during the worksheet setup).

warehouse: dbt_wh

database: dbt_demo_db

schema: marts

threads: 4

Test and Verify the Connection
dbt separates sensitive login credentials from the actual code folder. The connection details were automatically written to a global file on the Windows machine at C:\Users\<Windows_Username>\.dbt\profiles.yml.

C:\Users\ganes\.dbt\profiles.yml

Move your terminal into the newly created project folder:

Bash
cd my_snowflake_project
Run the debug check to test if VS Code can talk directly to the Snowflake cloud warehouse:

Bash
dbt debug
Look for a green success message at the bottom that says "All checks passed!". This confirms that the repository is linked, the files are ready, and your student's computer is successfully talking to Snowflake.
